웹 서버(윈도우)

아파치를 잘 쉽게 설치할 수 있도록 도와주는 프로그램들이 있다.
아파치 공식 홈페이지에 관련 내용이 나와있다. 그 중 하나로 bitnami wamp stack이 있다.
이 프로그램을 설치하면 자동으로 아파치가 설치되는 방식이다. 이 프로그램을 다운 받은 후
installer을 실행시킨다. 설치를 완료한 후 성공했는지 확인하기 위해서는
go to application이라는 버튼을 누르고, welcomerhk application창이 뜨면 성공한 것.
그리고 window 탐색기에서 bitnami를 설치한 폴더 중 manager-windows라는 프로그램을 실행시키면 웹 서버를 키고 끌 수 있다.

주소에 뜨는 /index.html은 내 컴퓨터 속 index.html이라는 파일을 불러온다는 의미이다.
주소 중 http는 인터넷 서비스 중 하나, 그리고 127.0.0.1등의 번호는 IP주소(컴퓨터 한 대를 가리키는 숫자)
근데 그 중에서 127.0.0.1은 웹 브라우저가 설치된 각자의 컴퓨터를 가리키는 주소이다.
따라서 http://127.0.0.1/index.html을 치면 설정한 대로 웹 서버가 서버 컴퓨터 내의 index.html을 찾아
웹 브라우저로 전송해주고 이를 해석해 화면에 띄우는 방식이다.
우리가 만든 파일들을 apache2폴더 속 htdocs폴더로 옮기면 우리가 운영하는 웹 서버가 우리가 만든 파일을 서비스하는 것이다.
단순히 컴퓨터 내의 파일을 여는 것과, 서버프로그램을 통해 인터넷으로 옮기는 것은 질적으로 완전히 다르다.
http를 이용하는 것은 서버프로그램에 요청하는 것, file://을 사용하는 것은 서버가 관여하지 않고,
웹브라우저가 직접 컴퓨터 내 파일을 읽는 것이다. http://은 hyper text transfer protocol이라는 뜻인데,
서로 다른 컴퓨터의 파일을 읽으려면 반드시 http를 이용해야 한다.


웹 서버와 웹 브라우저의 통신(윈도우)

두 대의 컴퓨터가 웹 페이지를 서로 주고받는 방법에 대해 알아보자.
서버 컴퓨터에 index.html 파일이 있는 상황이다. 이 때 웹 브라우저 컴퓨터가 서버를 통해 파일을 요청하려면
IP 주소가 필요하다. 마치 친구 집에 전화하는 것처럼. 스마트폰을 통해 노트북으로 만든 웹 서버 컴퓨터에 접속하면 된다.
웹 서버의 IP주소를 알아내는 방법. 네트워크 및 공유센터에서
연결이라 부르는 부분에서 Ehernet 또는 와이파이를 클릭하고 자세히 보기를 누르면 IPv4에 나와있는 주소가 IP주소이다.
cmd에 ipconfig라고 쳐도 나온다.
내 노트북 주소는 IPv4 주소: 192.168.137.1 이다.
서로 다른 컴퓨터로 파일을 불러오려 할 때 두 컴퓨터는 같은 네트워크 안에 있어야 한다.


수업을 마치며..

WEB1 수업이 코딩이 교양인 사람에게는 출구를, 직업인 사람에게는 입구가 되었으면 좋겠다.
웹 사이트를 아름답게 만들고 싶다면 CSS를 공부해야 한다.
만약 사용자와 상호작용하는 웹 페이지를 만들고 싶다면 Javascript를 공부해야 한다. 프론트앤드 가 해당.
만약 1억개의 웹페이지에서 공통된 코드들을 다른 것으로 바꿔야 한다면 1억개를 일일히 변경하긴 불가능하다.
이 때 이용하는 것이 백앤드 기술인데, 하나의 파일을 변경하면 1억개의 웹페이지가 동시에 바뀐다.
PHP, JSP, Node.js 등이 이런 기술이고, 이 기술에 종사하는 사람들을 백앤드 엔지니어라고 한다.
만약 웹사이트로 광고를 걸어 돈을 벌고싶다면 WEB2 광고를 배우면 된다.
